# ID Specification

This directory turns `ID` from a repo-organized workflow into a versioned protocol surface.

Goals:
- make protocol changes easier to track;
- separate evolving implementation/docs from versioned spec commitments;
- give contributors a clear place for compatibility and RFC work.
